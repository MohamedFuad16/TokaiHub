# Code Review Standards

## Goal
Provide a clear, actionable checklist for reviewers to evaluate PRs quickly and consistently.

## Rules

### 1. Review Focus
- Correctness
- Readability
- Maintainability
- Performance
- Security
- Accessibility
- Testing

### 2. Checklist
- Build and run tests locally
- Ensure tests cover new behavior
- Verify changes are isolated to the intended module
- Check naming, structure, and adherence to style
- No accidental secrets or credential leakage

### 3. Communication
- Leave constructive, solution-oriented feedback
- Suggest concrete follow-up tasks when needed

### 4. Documentation
- Update or add relevant docs if necessary
- Ensure code comments explain non-obvious decisions

### 5. Output Quality
- PR should be self-contained, with clear rationale and minimal noise
