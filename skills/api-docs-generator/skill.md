# API Docs Generator

## Goal
Produce structured API documentation from code annotations or OpenAPI-like specs, ready for onboarding and maintenance.

## Rules

### 1. Structure
Always structure into:
- Overview
- Endpoints
- Data Models
- Security
- Examples
- Implementation Notes

### 2. Writing Style
- Clear, precise, consistent
- Avoid ambiguities; use concrete examples
- Do not use emojis or conversational tone

### 3. Formatting
- Markdown with code blocks for examples and JSON
- Use bullet lists for readability

### 4. Enhancement
- Clarify ambiguous endpoints or data models
- Add missing details (headers, params, responses)

### 5. Output Quality
- Production-grade docs suitable for onboarding and external sharing
